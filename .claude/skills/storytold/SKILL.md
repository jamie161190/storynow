# Storytold: Product and Technical Skill

## What Storytold Is

Storytold (storytold.ai) is a personalised children's story product. A parent fills in details about their child, chooses a narrator voice, and receives a unique story generated by AI and read aloud by a professional voice. The 30-second preview plays free before payment so the customer is their own review.

**Live at:** storytold.netlify.app
**Domain:** storytold.ai (not yet pointed at Netlify)

## The Customer Flow

1. Landing page: "Create a story" or "Give as a gift"
2. Pick category: Bedtime, Journey, or Learning
3. Fill in child details: name, age, gender, best friend, interests, something they're proud of, optional pet, optional personal message
4. Choose story length: Short (~2min), Standard (~5min), Epic (~10min). Planned: 30min and 1hr premium tiers.
5. Choose narrator voice: 4 voices per category, matched to mood
6. Preview generates: 30 seconds of the story, free
7. Unlock full story: payment (Stripe, not yet built)
8. Delivered via web player link, saved to account library (not yet built)

## Categories

- **Bedtime**: Warm, calming, gentle. Voices are soft and soothing. Story winds down naturally, ends with child drifting to sleep.
- **Journey**: Exciting episodic adventure with cliffhangers. Voices are energetic. Designed for car rides, flights, trains.
- **Learning**: Child is a superhero who defeats a villain using a school subject (maths, science, geography, history, phonics). Voices are encouraging teacher types.

## Key Product Decisions

- **No voice cloning.** Tested with ElevenLabs, quality was not accurate enough across accents. Parked. Launching with narrator voices only.
- **No PDF.** The product is an audio experience delivered via web player, not a document.
- **Preview before payment.** This eliminates the trust problem. Nobody buys blind. No reviews needed at launch.
- **Personal message at start.** Parent's words ("Chase, this is from Daddy...") read aloud before the story. One of the most powerful features.

## Design Direction

Jamie has chosen **playful and colourful**. Think CBeebies meets a premium children's brand. Bright, fun, illustration-led, but polished enough to feel worth paying for.

Current design uses warm/cream tones (Playfair Display + Inter fonts). This needs updating to match the playful direction: soft purples, warm oranges, sky blues, rounded shapes, generous whitespace. Joyful, not corporate.

**Logo:** `story` in deep colour, `told` in accent colour. Simple text mark.

**Brand voice:** Warm but simple. Emotional without being sentimental. Speak like a parent who built this for their own child. Never use corporate language. Use "we" for the product, "you/your child/their" for the customer.

## Technical Architecture

### File Structure
```
storytold/
  public/index.html              # Entire frontend (single page app, vanilla JS)
  netlify/functions/
    generate-preview.mjs          # Claude story generation + ElevenLabs preview audio
    generate-full.mjs             # Full story audio via ElevenLabs
    voices.mjs                    # Voice options per category
  netlify.toml                    # Build config, API redirects
  deploy.sh                       # One-command deploy script
  .env                            # Local dev keys (not in git)
```

### Stack
- **Hosting:** Netlify (serverless functions + static)
- **Frontend:** Single HTML file, vanilla JS, no framework
- **Story AI:** Anthropic Claude Sonnet via API
- **Voice:** ElevenLabs Text-to-Speech (Starter plan, $5/mo)
- **Payments:** Stripe (planned)

### API Flow
Frontend POSTs to `/api/generate-preview` with storyData + voiceId. Function calls Claude for story text, sends first 40 words to ElevenLabs for preview audio, returns base64 audio. On unlock, `/api/generate-full` sends full text to ElevenLabs.

### Environment Variables (Netlify)
- `ANTHROPIC_API_KEY`
- `ELEVENLABS_API_KEY`

### Voice IDs
Hardcoded per category in both `voices.mjs` and the VOICES object in `index.html`. When changing voices, update both files.
